# GPX Viewer · GPX 轨迹分析工具

在浏览器中打开 GPX 文件，在地图上查看、回放与分析轨迹。**无需安装**，数据在本地解析，不上传服务器。

**在线体验：** [https://gpxviewer.github.io/gpxviewer/](https://gpxviewer.github.io/gpxviewer/)

仓库地址：[github.com/gpxviewer/gpxviewer](https://github.com/gpxviewer/gpxviewer)

---

## 亮点概览

| 类别 | 能力 |
|------|------|
| 地图 | 5 种底图，自动处理 WGS-84 / GCJ-02 / BD-09 坐标 |
| 可视化 | 普通轨迹、速度热力、海拔热力；速度/海拔剖面曲线 |
| 分析 | 区间对比、停靠点、里程牌、速度/海拔里程分布图 |

| 体验 | 单文件部署、离线可用、手机竖屏适配 |

## 核心特性

### 多文件轨迹管理

- 一次导入多个 `.gpx` 文件，侧边栏列表切换
- 每条轨迹自动生成 **Canvas 缩略图**，快速辨认路线形状
- 支持上一条 / 下一条轨迹切换

### 多底图与坐标系

内置五种底图，切换时自动匹配坐标系，避免轨迹偏移：

- OpenStreetMap（WGS-84）
- 高德地图、腾讯地图（GCJ-02）
- 谷歌地图（WGS-84）
- 百度地图（BD-09）

Leaflet 与瓦片资源 **打包在 `vendor/` 目录**，不依赖国外 CDN，国内访问更稳定。

### 三种轨迹渲染模式

循环切换：

1. **普通模式** — 统一蓝色轨迹线  
2. **速度热力** — 绿→红表示慢→快（km/h）  
3. **海拔热力** — 绿→红表示低→高（m）

底部 **速度/海拔曲线图** 与地图模式联动；点击曲线可跳转到对应里程位置。轨迹点过多时自动降采样，保证流畅。

### 精细回放与控制

- 底部 **进度滑块** 沿轨迹里程拖动
- 步进距离：1 m / 10 m / 100 m / 1 km / 10 km，支持长按连续步进
- **播放** 按钮自动沿轨迹前进，到终点自动停止
- 曲线图可开启 **「区间」** 模式：对比「上一点 → 当前点」，显示行驶距离、海拔差、坡度、用时、均速等；地图与曲线同步高亮 A/B 区间

### 智能信息面板

左上角实时显示：

- 时间、速度、累计距离、经纬度
- **当前海拔** 文字颜色与海拔热力图一致（低绿高红）
- 点击 **当前速度 / 当前海拔** 打开 **里程分布柱状图**：统计各速度或海拔区间内累计行驶了多少公里（整数步进、最多 15 段，悬停/点击柱子查看详情）

点击经纬度可打开 **导航面板**：在 WGS-84、GCJ-02、BD-09 间切换，并跳转高德 / 百度 / 腾讯 / 谷歌 / 系统地图或复制坐标。

### 地图标注与分析

- **最高 / 最低海拔**、**最快速度** 自动标点  
- **停靠点**：在约 1 km 范围内停留达到阈值（默认 30 分钟）的地点；位置按 **停留时长加权** 计算，而非简单平均  
- **里程牌**：按 1 / 2 / 5 / 10 … km 整数公里立牌，间隔随缩放与视口 **自动调节**，避免过密或过疏  

### 海拔数据清洗

对气压计 / GPS 异常跳变进行过滤与插值（尖峰检测、垂直速率、大时间间隔跳变等），让热力图与剖面更可信。参数可在 `index.html` 顶部常量区调整（如航空轨迹需放宽垂直速率上限）。

### 地图视图三态（复位按钮）

连续点击 **复位地图** 按钮循环：

1. 定位到当前点  
2. **锁定** 当前点在地图中心（拖动进度时跟随）  
3. 缩放到整段轨迹全览  

手动拖地图会退出锁定序列。

### 移动端适配

竖屏窄屏时，底部 **地图 / 模式 / 复位** 三个按钮自动换到第二行，避免与播放、步进控件挤在一行溢出。

## 快速开始

### 在线使用

1. 打开 [https://gpxviewer.github.io/gpxviewer/](https://gpxviewer.github.io/gpxviewer/)  
2. 点击右上角菜单 → **导入 GPX**（或选择文件）  
3. 在列表中选中轨迹，使用底部滑块与按钮浏览  

### 本地运行

```bash
git clone https://github.com/gpxviewer/gpxviewer.git
cd gpxviewer
# 任意静态服务器，例如：
python3 -m http.server 8080
# 浏览器访问 http://localhost:8080
```

也可直接用浏览器打开 `index.html`（部分环境下文件协议可能影响地图瓦片加载，推荐本地 HTTP 服务）。

### 部署到 GitHub Pages

将仓库 `main` 分支发布到 Pages 即可；站点根目录需包含 `index.html` 与 `vendor/`（当前仓库结构已满足）。

## 技术说明

- **单页应用**：全部逻辑在 `index.html` 中，无构建步骤  
- **地图**：Leaflet 1.9.4（本地 vendor）  
- **GPX 解析**：浏览器端 `DOMParser`，支持 `trkpt` 的时间、海拔、速度标签  
- **速度**：优先读取 GPX 内 `speed`；缺失时由相邻点距离与时间推算；自动识别 m/s 与 km/h  
- **可配置常量**：步进阈值、停靠判定、海拔清洗、柱状图步进、里程牌密度等均在 `index.html` 内用 `const` 标注注释，便于按需修改
